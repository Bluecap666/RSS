# 📥 RSS 源导入导出功能使用指南

## ✨ 功能概述

提供完整的 RSS 源备份和恢复功能，支持数据的导入和导出。

**主要特点**:
- ✅ 一键导出所有 RSS 源配置
- ✅ 从备份文件导入数据
- ✅ JSON 格式，易于编辑和分享
- ✅ 自动验证数据格式
- ✅ 支持批量导入

## 🎯 使用场景

### 导出数据
- 📱 更换设备时迁移数据
- 💾 定期备份 RSS 源列表
- 📤 分享给其他人相同的订阅源
- 🔧 保存当前配置状态

### 导入数据
- 📲 新设备恢复订阅列表
- 🔄 从备份中恢复数据
- 📥 导入他人分享的订阅源
- 🔙 回滚到之前的配置

## 📖 使用方法

### 方式一：通过侧边栏操作

#### 导出步骤

1. **打开侧边栏**
   - 桌面端：点击左上角菜单按钮 ☰
   - 移动端：点击左上角菜单按钮 ☰

2. **点击"导入导出"按钮**
   - 位于侧边栏底部
   - 图标：📤 (分享图标)

3. **选择"导出数据"**
   - 点击紫色渐变的"导出数据"按钮
   - 系统自动生成 JSON 备份文件
   - 文件命名：`rss-backup-YYYY-MM-DD.json`

4. **保存文件**
   - 浏览器会自动下载文件
   - 建议保存到安全位置
   - 可以重命名为更有意义的名称

#### 导入步骤

1. **打开侧边栏**

2. **点击"导入导出"按钮**

3. **选择"导入数据"**
   - 点击粉色渐变的"导入数据"按钮
   - 系统弹出文件选择对话框

4. **选择备份文件**
   - 选择之前导出的 `.json` 文件
   - 必须是有效的 RSS 备份格式

5. **确认导入**
   - 系统自动验证文件格式
   - 验证成功后显示提示
   - 页面自动刷新，加载新数据

### 方式二：手动操作 (高级)

#### 手动导出

```javascript
// 在浏览器控制台执行
import storage from './src/services/storage'
storage.exportData()
```

#### 手动导入

```javascript
// 读取 JSON 文件内容后执行
const jsonString = `你的 JSON 数据`
storage.importData(jsonString)
location.reload()
```

## 📋 数据格式

### 导出的 JSON 文件结构

```json
{
  "version": "1.0",
  "exportDate": "2026-03-17T12:00:00.000Z",
  "feeds": [
    {
      "id": "1234567890",
      "title": "RSS 源标题",
      "url": "https://example.com/rss",
      "description": "可选的描述信息",
      "lastUpdate": 1234567890,
      "items": []
    }
  ],
  "settings": {
    // 用户设置
  }
}
```

### 字段说明

**必填字段**:
- `feeds`: RSS 源数组
  - `id`: 唯一标识符
  - `title`: RSS 源标题
  - `url`: RSS 源地址

**可选字段**:
- `version`: 数据格式版本
- `exportDate`: 导出时间
- `settings`: 用户设置
- `items`: 文章列表 (通常为空，不导出文章详情)

## ⚠️ 注意事项

### 导出数据

1. **数据完整性**
   - 导出包含所有 RSS 源配置
   - 不包含文章详情 (节省空间)
   - 包含基本设置

2. **文件大小**
   - 通常很小 (< 1MB)
   - 只包含配置信息
   - 适合云端存储

3. **隐私考虑**
   - 备份文件包含个人订阅习惯
   - 不要随意分享给陌生人
   - 妥善保存备份文件

### 导入数据

1. **数据来源**
   - ✅ 自己导出的备份文件
   - ✅ 信任的人分享的订阅源
   - ❌ 来源不明的文件

2. **覆盖规则**
   - 导入会**完全覆盖**现有数据
   - 现有 RSS 源会被替换
   - 建议先导出当前数据

3. **兼容性**
   - 仅支持标准 JSON 格式
   - 需要符合指定的数据结构
   - 不支持 OPML 格式 (计划中)

## 🔄 最佳实践

### 定期备份

**推荐频率**:
- 📅 每月导出一次
- ➕ 添加重要 RSS 源后立即备份
- 🔄 重大调整后备份

**存储建议**:
- 💾 云盘同步文件夹
- 📧 发送给自己作为邮件附件
- 🔐 加密存储敏感数据

### 版本管理

**命名建议**:
```
rss-backup-2026-03-17.json          // 日期版本
rss-backup-2026-03-before-move.json // 事件标记
rss-feeds-latest.json               // 最新版本
```

**多版本保留**:
- 保留最近 3-5 个备份
- 删除过旧的版本
- 标注重要变更点

## 🆘 故障排除

### 问题 1: 导入失败

**可能原因**:
- JSON 文件格式不正确
- 文件被修改或损坏
- 缺少必需字段

**解决方法**:
1. 检查文件是否为有效 JSON
2. 使用 JSON 验证工具验证
3. 重新导出原始备份

### 问题 2: 导出后找不到文件

**可能原因**:
- 浏览器阻止了下载
- 下载目录不明确
- 文件名被修改

**解决方法**:
1. 检查浏览器下载记录
2. 查看默认下载文件夹
3. 重新导出并注意保存位置

### 问题 3: 导入后数据丢失

**可能原因**:
- 备份文件本身有问题
- 导入过程被中断
- 浏览器缓存问题

**解决方法**:
1. 尝试其他备份文件
2. 清除浏览器缓存后重试
3. 检查备份文件的 exportDate

可以用文本编辑器打开 JSON 文件:

**用途**:

### 合并多个备份

可以手动合并多个 JSON 文件:

```javascript
// 合并两个备份
const backup1 = require('./backup-1.json')
const backup2 = require('./backup-2.json')

const merged = {
  ...backup1,
  feeds: [...backup1.feeds, ...backup2.feeds]
}
```
